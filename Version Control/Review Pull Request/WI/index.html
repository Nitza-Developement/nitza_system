Guía de revisión de Pull Requests usando VSCode

Requisitos previos

  • VSCode instalado
  • Extensión GitHub Pull Requests and Issues instalada
  • Cuenta de GitHub configurada en VSCode
  • Git instalado y configurado localmente

1. Revisión inicial de documentación

  1. Verifica que el PR incluya:
  2. Descripción detallada de cambios
  3. Capturas de pantalla/grabaciones para cambios visuales
  4. Pasos para probar los cambios
  5. Enlaces a tareas en odoo relacionadas

2. Checkout y prueba local

  1. En VSCode, ve a la sección GitHub Pull Requests
  2. Haz clic derecho en el PR > "Checkout Pull Request"

  1. Verifica que los cambios se descargaron:

3. Verificación independiente

  1. Sigue los pasos de prueba documentados
  2. Para cambios visuales:
  3. Compara con las capturas proporcionadas
  4. Prueba en diferentes resoluciones
  5. Verifica estados de hover/focus
  6. Para cambios funcionales:
  7. Ejecuta pruebas unitarias
  8. Prueba casos límite
  9. Verifica integración con otros componentes

4. Revisión de código

  1. En la vista "Changes":
  2. Revisa la calidad del código
  3. Verifica convenciones del proyecto

  4. Busca problemas de seguridad

  5. Agrega comentarios:

  6. Hover sobre la línea > ícono +

  7. Para múltiples líneas: selecciona > clic derecho > "Add Comment"

5. Acciones finales

  1. Aprobar:
  2. "Add Review" > "Approve"

  3. Confirma que probaste los cambios localmente

  4. Solicitar cambios:

  5. "Add Review" > "Request Changes"
  6. Detalla qué necesita modificación
  7. Incluye evidencia de problemas encontrados

6. Limpieza

  1. Al aprpobar cambios elimina la rama del "Pull Request"

  1. Asegúrate de volver a tener la rama dev actualizada

Mejores prácticas

  • No apruebes sin probar localmente
  • Documenta todos los problemas encontrados
  • Verifica que los cambios coincidan con la documentación
  • Prueba en un entorno limpio
